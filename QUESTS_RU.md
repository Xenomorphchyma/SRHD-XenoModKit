# Текстовые квесты QM/QMM в SRHD ModKit 0.9.1

ModKit работает с текстовыми квестами полностью headless и не требует TGE.
Поддерживается чтение QM 2/3/4 и QMM 6/7, редактируемый JSON и сборка QMM 7.
Исходный файл не изменяется ни одной командой чтения или проверки.

## Возможности

- безопасный разбор заголовка, строк UTF-16LE, параметров, локаций и переходов;
- ограничения на размеры и счётчики до выделения памяти;
- проверка идентификаторов и ссылок графа;
- поиск недостижимых локаций и потенциальных циклов автоматических переходов;
- проверка диапазонов параметров, условий и деления по модулю;
- разбор формул с `[p1]`, диапазонами `[1..4;8]`, `mod`, `div`, `to`, `in`,
  `and`, `or`, сравнением и арифметикой;
- список изображений, звуков и музыкальных дорожек, на которые ссылается квест;
- детерминированная сборка QMM 7 и обязательный смысловой round-trip;
- единый аудит `text-quests`: быстрый parse/validate в `dev` и полный
  `QM/QMM → QMM → модель` в `release`.

## Проверка готового квеста

```powershell
python -B srhd.py quest info "D:\work\Quest.qmm" --json
python -B srhd.py quest validate "D:\work\Quest.qmm" --json
python -B srhd.py quest roundtrip "D:\work\Quest.qmm" --json
```

`info` сообщает версию, количество параметров, локаций и переходов, найденные
медиа и проблемы модели. `validate` возвращает код `2`, если найдена ошибка.
`roundtrip` собирает QMM только во временной папке, перечитывает его и сравнивает
игровую модель; временный файл не публикуется.

Пустая формула изменения в старых квестах допустима и означает отсутствие
операции. Она отмечается предупреждением, но не блокирует сборку. Синтаксически
повреждённая непустая формула или ссылка на отсутствующий параметр является
ошибкой.

## Безопасное редактирование

```powershell
python -B srhd.py quest export-json "D:\work\Quest.qmm" "D:\work\Quest.json"
# Изменить Quest.json обычным текстовым редактором или программно.
python -B srhd.py quest build "D:\work\Quest.json" "D:\work\Quest.edited.qmm" --json
python -B srhd.py quest validate "D:\work\Quest.edited.qmm" --json
```

JSON имеет схему `srhd-modkit-quest-v1`. Верхний уровень содержит метаданные
формата, настройки квеста, `parameters`, `strings`, `success_text`, `task_text`,
`locations` и `jumps`. Медиа хранится как `img`, `sound`, `track`. Индексы
массивов изменений и условий соответствуют параметрам квеста; их нельзя
укорачивать произвольно.

`quest build`:

1. загружает JSON и проверяет имя схемы;
2. строит объектную модель и выполняет смысловую валидацию;
3. блокирует публикацию при ошибках;
4. кодирует детерминированный QMM 7 во временный файл рядом с назначением;
5. перечитывает файл и сравнивает модель;
6. атомарно публикует результат только после успешной проверки.

Существующий output не заменяется без `--overwrite`. Для работы агента лучше
указывать новое имя, проверить его и только затем отдельно решать вопрос о
замене рабочей копии.

Старый QM при сборке обновляется до QMM 7. Поля `Parsec` и `Artefact`, которые
старый формат хранил как подписи подстановок, в QMM не сериализуются и поэтому
не входят в смысловое сравнение. Остальное игровое состояние должно совпасть.

## Python API

```python
from srhd_modkit import (
    build_quest_from_json,
    export_quest_json,
    inspect_quest,
    load_quest,
    quest_media,
    validate_quest,
    verify_quest,
    write_qmm,
)

document = load_quest(r"D:\work\Quest.qmm")
issues = validate_quest(document)
print(quest_media(document))
print(verify_quest(r"D:\work\Quest.qmm"))
```

Низкоуровневый `write_qmm()` принимает модель и пишет поток QMM. Для обычной
автоматизации предпочтительнее `build_quest_from_json()`: он выполняет
валидацию, временную публикацию и круговую проверку.

## Что ещё проверяется только в игре

- фактическое прохождение всех веток и отображение интерфейса;
- наличие всех медиа в выбранной установке и корректность их содержимого;
- интеграция выдачи/завершения квеста со скриптами, сохранениями и другими модами;
- баланс формул, текстов и вероятностей.

Форматная и графовая валидность не является обещанием корректного игрового
сценария. Перед релизом всё равно требуется тестовый запуск в Space Rangers HD.
